# 6.3.1 Web 安全概述

## 1. 概述

Web 安全是保护 Web 应用免受各种安全威胁的实践。Web 应用面临多种安全威胁，需要实施多层防护措施来保护应用和用户数据。

## 2. 核心威胁

### 2.1 常见威胁

- **XSS（跨站脚本攻击）**：注入恶意脚本
- **CSRF（跨站请求伪造）**：伪造用户请求
- **SQL 注入**：注入恶意 SQL 语句
- **身份认证绕过**：绕过身份认证
- **敏感数据泄露**：泄露敏感信息

### 2.2 威胁分类

- **注入攻击**：SQL 注入、命令注入、XSS
- **身份认证缺陷**：弱密码、会话固定
- **敏感数据暴露**：明文存储、传输加密不足
- **XML 外部实体（XXE）**：XML 解析漏洞
- **失效的访问控制**：权限绕过
- **安全配置错误**：默认配置、错误配置
- **XSS**：存储型、反射型、DOM 型
- **不安全的反序列化**：反序列化漏洞
- **使用含有已知漏洞的组件**：依赖漏洞
- **日志和监控不足**：缺乏安全审计

## 3. 防护原则

### 3.1 纵深防御

- 多层防护措施
- 不依赖单一防护
- 防御失效时的备用方案
- 持续监控和更新

### 3.2 最小权限

- 最小权限原则
- 默认拒绝访问
- 明确授予权限
- 定期审查权限

### 3.3 输入验证

- 验证所有输入
- 清理用户输入
- 使用白名单验证
- 避免黑名单验证

## 4. 安全最佳实践

### 4.1 开发阶段

- 安全编码规范
- 代码审查
- 安全测试
- 依赖管理

### 4.2 部署阶段

- 安全配置
- HTTPS 配置
- 安全头设置
- 日志和监控

### 4.3 运维阶段

- 定期更新
- 安全监控
- 漏洞扫描
- 事件响应

## 5. 注意事项

- **安全意识**：提高安全意识
- **持续更新**：关注安全漏洞和最佳实践
- **多层防护**：实现多层安全防护
- **安全测试**：定期进行安全测试

## 6. 常见问题

### 6.1 如何评估应用安全性？

使用安全扫描工具、渗透测试、代码审查。

### 6.2 如何处理安全漏洞？

及时修复漏洞、发布安全补丁、通知用户。

### 6.3 如何建立安全流程？

制定安全规范、实施安全测试、建立响应流程。
