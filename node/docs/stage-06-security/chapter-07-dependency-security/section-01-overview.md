# 6.7.1 依赖安全概述

## 1. 概述

依赖安全是保护应用免受第三方依赖漏洞影响的重要措施。现代应用大量使用第三方依赖，依赖漏洞可能成为安全风险的主要来源。

## 2. 依赖安全威胁

### 2.1 常见威胁

- **已知漏洞**：依赖包含已知安全漏洞
- **恶意包**：依赖包含恶意代码
- **供应链攻击**：通过依赖链传播攻击
- **过时依赖**：使用过时的不安全版本

### 2.2 风险来源

- **直接依赖**：直接安装的包
- **间接依赖**：依赖的依赖
- **开发依赖**：开发工具依赖
- **全局依赖**：全局安装的包

## 3. 依赖安全措施

### 3.1 漏洞检测

- **npm audit**：npm 内置漏洞检测
- **Snyk**：第三方漏洞检测工具
- **Dependabot**：GitHub 自动依赖更新
- **OWASP Dependency-Check**：OWASP 依赖检查工具

### 3.2 依赖管理

- **版本锁定**：使用 package-lock.json
- **定期更新**：定期更新依赖
- **最小依赖**：只安装必要的依赖
- **依赖审查**：审查依赖来源

## 4. 最佳实践

### 4.1 依赖选择

- 选择维护活跃的包
- 检查包的安全性
- 审查包的历史
- 使用官方包

### 4.2 依赖管理

- 定期更新依赖
- 监控依赖漏洞
- 及时修复漏洞
- 记录依赖变更

## 5. 注意事项

- **及时更新**：及时更新有漏洞的依赖
- **测试验证**：更新后测试应用
- **版本兼容**：注意版本兼容性
- **安全审查**：定期审查依赖

## 6. 常见问题

### 6.1 如何选择安全的依赖？

选择维护活跃、安全记录良好的包，检查包的历史和社区。

### 6.2 如何处理间接依赖漏洞？

更新直接依赖，或使用 npm audit fix 自动修复。

### 6.3 如何防止供应链攻击？

审查依赖来源、使用锁定文件、定期安全扫描。
