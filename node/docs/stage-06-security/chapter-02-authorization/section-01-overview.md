# 6.2.1 授权概述

## 1. 概述

授权（Authorization）是验证用户权限的过程，确定用户是否有权限执行特定操作或访问特定资源。授权是应用安全的重要组成部分，确保用户只能访问其有权限的资源。

## 2. 核心概念

### 2.1 授权 vs 认证

- **认证（Authentication）**：验证用户身份（你是谁）
- **授权（Authorization）**：验证用户权限（你能做什么）

### 2.2 授权模型

- **RBAC（基于角色的访问控制）**：通过角色分配权限
- **ABAC（基于属性的访问控制）**：通过属性判断权限
- **ACL（访问控制列表）**：直接分配权限
- **DAC（自主访问控制）**：资源所有者控制权限

## 3. 授权流程

### 3.1 基本流程

```
1. 用户请求资源
2. 系统验证用户身份（认证）
3. 系统检查用户权限（授权）
4. 权限验证通过，允许访问
5. 权限验证失败，拒绝访问
```

### 3.2 权限检查点

- **路由级别**：在路由中间件中检查
- **控制器级别**：在控制器中检查
- **数据级别**：在数据访问层检查
- **UI 级别**：在用户界面中检查

## 4. 授权策略

### 4.1 最小权限原则

- 用户只获得必要的权限
- 默认拒绝访问
- 明确授予权限
- 定期审查权限

### 4.2 权限继承

- 角色继承权限
- 组继承权限
- 层级权限继承

## 5. 最佳实践

### 5.1 权限设计

- 清晰的权限模型
- 细粒度权限控制
- 权限可扩展
- 权限可审计

### 5.2 安全考虑

- 服务器端验证
- 防止权限提升
- 记录权限操作
- 定期权限审查

## 6. 注意事项

- **服务器验证**：始终在服务器端验证权限
- **最小权限**：遵循最小权限原则
- **权限缓存**：合理缓存权限信息
- **权限审计**：记录权限操作

## 7. 常见问题

### 7.1 如何选择授权模型？

根据应用复杂度、权限粒度、扩展性需求选择。

### 7.2 如何处理权限缓存？

使用 Redis 缓存权限信息，设置合理的过期时间。

### 7.3 如何实现权限继承？

使用树形结构或图结构实现权限继承。
