# 1.4 TypeScript 工程化环境配置

## 1. 概述

本章解析如何在 Node.js 生态中构建工业级的 TypeScript 开发工程。重点涵盖基于 ES Modules (ESM) 的项目初始化逻辑、`tsconfig.json` 的核心编译策略、高性能运行时工具（tsx）的应用以及生产环境的构建与 Source Map 追踪。掌握本章内容是实现代码从“开发阶段”向“生产部署”平滑过渡的工程化保障。

| **1.4.4**| [生产环境构建与运行](section-04-build-run.md)                   | 编译流水线、Source Map 追踪、生产环境优化运行。  |

## 3. 学习目标

- **工程构建**：能够独立搭建并运行符合 ESM 与 TypeScript 规范的 Node.js 后端项目。
- **配置优化**：深刻理解编译选项对代码安全性、包兼容性及构建速度的影响。
- **排障能力**：掌握在生产环境下通过 Source Map 精准定位 TypeScript 源码错误的方法。
