## 核心概念

- **国际化（i18n）**：使应用程序能够适应不同语言和地区
- **本地化（l10n）**：为特定地区定制应用程序
- **区域设置（Locale）**：语言和地区的标识符（如 'zh-CN'、'en-US'）
- **Intl API**：JavaScript 的国际化 API，提供日期、数字、货币等格式化功能

## 学习建议

1. **理解区域设置**：重点理解区域设置标识符的格式
2. **实践应用**：通过实际项目练习国际化格式化
3. **注意格式差异**：不同地区的日期、数字格式可能不同
4. **性能考虑**：Intl API 的性能考虑和优化
